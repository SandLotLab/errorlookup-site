429 Too Many Requests vs 503 Service Unavailable

429 is client-specific throttling; 503 is service-wide unavailability.

Aspect429 Too Many Requests503 Service Unavailable
When it happensPrimary condition specific to 429 too many requests.Primary condition specific to 503 service unavailable.
Troubleshooting first stepCheck logs and config where 429 is generated.Check logs and config where 503 is generated.
Client impactDifferent retry behavior expected.Different retry behavior expected.

When each one appears

Use response semantics accurately so clients, CDNs, and crawlers make correct decisions.

How to troubleshoot each

Trace request through edge, gateway, app, and data layers. Confirm whether failure is auth, policy, overload, upstream timeout, or origin crash.
